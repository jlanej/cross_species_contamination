2. Methods

2.1 Sample processing and classification

  1. Extract (csc-extract): streaming extraction of unmapped reads (and optionally low-MAPQ reads) from aligned BAM/CRAM files using samtools. Per-sample totals from samtools idxstats are captured in a reads_summary.json sidecar before any downstream filtering so they can serve as the authoritative absolute-burden denominator.
  2. Classify (csc-classify): taxonomic assignment of the extracted reads with Kraken2 against a curated reference database. The recommended database is PrackenDB (one NCBI reference genome per species for bacteria, archaea, fungi, protists, viruses, human, and UniVec Core).
  3. Aggregate (csc-aggregate): per-taxon read counts are collated across samples into three matrices with distinct, explicitly labelled denominators (see §2.2). Rank-filtered matrices (S, G, F) are emitted alongside the unfiltered matrices.
  4. Detect (csc-detect, optional): statistical outlier detection using MAD, IQR, and/or a two-component GMM to flag samples whose profile deviates from the cohort.

2.4 Outlier-detection parameters

csc-detect applies statistical outlier detection (MAD, IQR, and/or a two-component GMM) per taxon, flagging samples whose abundance exceeds the cohort distribution. The detector identifies upper-tail outliers only; under-representation is not informative for contamination. The same parameters are applied to every matrix variant analysed (rank-filtered matrices, confidence-tier matrices, and the absolute-burden side pass).

method"all"
matrix_type"cpm"

2.5 Absolute-burden side pass

In addition to the primary detection pass, csc-detect automatically ran a parallel pass on taxa_matrix_abs.tsv (reads per million total sequenced reads, from samtools idxstats). The compositional CPM denominator is invariant to how many reads were classified, so two libraries with very different host-depletion efficiencies can produce indistinguishable CPM profiles even when their absolute non-human burdens differ by orders of magnitude. The absolute-burden pass uses a denominator that is independent of host depletion and therefore catches contamination episodes the CPM pass can mask. Results are written to detect/abs/ and are summarised side-by-side with the primary pass in §3.3.

7. Discussion & Caveats

  • Kraken2 assignments are only as reliable as the reference database; use a non-redundant database (e.g. PrackenDB) where possible to avoid LCA inflation.
  • Compositional metrics (CPM, stacked bars, diversity) describe the shape of the non-human fraction but are invariant to its size. Always pair CPM with absolute burden (§4) when available.
  • Laboratory and reagent contaminants ('kitome') frequently appear as core, low-burden species in §3.2; consider adding them to csc-detect --kitome-taxa.
  • Per-sample handling errors typically appear as outburst species (low prevalence, very high absolute burden in a handful of samples) — cross-reference §3.2 with §4.
  • Core species in this cohort: root, Bacteria, unclassified. Their high prevalence at low-to-moderate burden is the fingerprint of reagent / library contamination (kitome). Consider whether they should be excluded from any biological interpretation or added to a kitome ignore list.
  • Outburst species: Archaea, Methanobrevibacter smithii, artificial sequences. Rare across the cohort but with substantial burden where they appear — investigate the affected samples (§3.8 / §6) for handling errors or sample-specific contamination.
  • 20 sample(s) were flagged in §4 as potentially impacting variant calling. These should be inspected manually before downstream use.
  • 1 sample(s) were flagged only by the absolute-burden side pass — characteristic of variable host-depletion efficiency rather than a single-organism contaminant.
